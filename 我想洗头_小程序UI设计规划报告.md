## 1. 设计目标与原则

### 1.1 产品定位

面向 **住院患者及家属** 的上门洗头 O2O 服务平台，通过微信小程序连接用户与专业洗头师傅。

### 1.2 设计目标

| 维度 | 目标 | 衡量标准 |
|------|------|---------|
| **信任感** | 用户第一次打开就觉得专业、可靠、有温度 | 新用户首页停留时长 ≥ 5s，首单转化率 ≥ 15% |
| **极简操作** | 从打开小程序到完成下单 ≤ 3 分钟，操作步骤 ≤ 5 步 | AC-01 验收标准 |
| **温度感** | 品牌调性是"关怀与体面"，而非冰冷的交易撮合 | 用户评价中"有温度"相关关键词出现率 |
| **医院场景适配** | 信息架构和文案风格贴合医疗护理场景 | 医院场景下单占比 ≥ 60% |

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **信任优先** | 每一次交互都要传递专业、安全、可信赖——实名认证标识、平台保险徽章、评价体系前置展示 |
| **一步到位** | 目标用户（患者家属）处于焦虑状态，减少认知负荷，核心路径不跳出当前页面层级超过 3 级 |
| **防呆设计** | 关键操作有二次确认，表单有实时校验，错误状态有明确引导 |
| **包容性设计** | 考虑中老年用户的字体大小、按钮区域、对比度；考虑病房环境的光线、噪音、操作姿势 |
| **渐进呈现** | 首屏只展示最核心信息，次要功能折叠在二级入口，不信息过载 |

## 2. 品牌视觉定位

### 2.1 品牌调性

```
专业  ──────────────  温暖
  医疗蓝（信任）      +    暖橙（关怀）
  #1A3A5C             +    #F08C4E
```

### 2.2 情绪板关键词

- **核心**：专业、洁净、关怀、体面
- **辅助**：温和、安静、可靠、细致
- **避免**：冰冷、廉价、娱乐化、过度促销化

### 2.3 色彩建议

基于已有原型中的配色方案，建议延续并规范化：

| 色名 | 色值 | 用途 | 色阶建议 |
|------|------|------|---------|
| **深蓝（主色）** | `#1A3A5C` | 主色调、品牌色、导航栏、按钮主色 | 50/100/200/300/400/500/600/700 |
| **亮蓝（辅助）** | `#2A5A8C` | 渐变辅助色、卡片头部背景 | - |
| **信息蓝** | `#0984E3` | 链接、信息提示、师傅级别标识 | - |
| **暖橙（点缀色）** | `#F08C4E` | CTA 按钮、价格高亮、新客标签、活跃状态 | 50/100/200/300/400/500 |
| **深橙** | `#E17055` | 价格文字、紧急状态、警告 | - |
| **翠绿** | `#00B894` | 完成状态、在线状态、安全确认 | - |
| **青绿** | `#00A381` | 绿色按钮悬停/按压 | - |
| **文字主色** | `#2D3436` | 正文 | - |
| **文字辅助** | `#636E72` | 次要文字、说明 | - |
| **文字弱化** | `#B2BEC3` | 占位符、禁用文字 | - |
| **背景色** | `#F5F6F8` | 页面背景 | - |
| **卡片背景** | `#FFFFFF` | 卡片、列表项、表单 | - |
| **分割线** | `#EDEDED` / `#F0F4F8` | 分割线、描边 | - |

### 2.4 字体建议

| 层级 | 字号 | 字重 | 颜色 | 用途 |
|------|------|------|------|------|
| H1 | 22px / 20px | Bold (700) | #1A3A5C | 页面标题、Banner 主文案 |
| H2 | 17px / 16px | SemiBold (600) | #1A3A5C | 模块标题、师傅名称 |
| H3 | 15px / 14px | SemiBold (600) | #2D3436 | 卡片标题、订单金额 |
| Body | 14px / 13px | Regular (400) | #2D3436 | 正文 |
| Body-Small | 12px | Regular (400) | #636E72 | 辅助文字、标签 |
| Caption | 11px / 10px | Regular (400) | #B2BEC3 | 时间、次要信息 |
| Price | 18px / 22px | Bold (700) | #E17055 | 价格展示 |
| Button | 14px / 16px | SemiBold (600) | - | 按钮文字 |

> **说明**：微信小程序默认字体为 `-apple-system, "PingFang SC", "Helvetica Neue", sans-serif`，建议统一使用系统字体，无需额外加载字体文件。

### 4.2 页面状态矩阵

每个页面需覆盖以下状态：

| 状态 | 说明 | 适用场景 |
|------|------|---------|
| **默认态** | 正常数据展示 | 所有页面 |
| **加载中** | Skeleton / Loading 动画 | 所有包含异步数据的页面 |
| **空状态** | 无数据时的引导 | 订单列表（某Tab为空）、收藏列表、搜索无结果、无师傅城市 |
| **错误态** | 网络异常/服务器错误 | 所有需要网络请求的页面 |
| **边界态** | 数据极值情况 | 师傅列表（几百条）、评价（仅1条/0条）、师傅约满 |
| **权限态** | 未授权/无权限 | 定位拒绝、手机号拒绝 |

## 5. 核心交互流程

### 5.1 用户首次打开 → 完成首单（黄金路径）

```
打开小程序
  │
  ▼
[定位授权弹窗]
  ├── 授权 → 自动定位到当前城市 → 首页
  └── 拒绝 → 默认"全国" → 提示手动选择城市 → 城市选择页
  │
  ▼
[首页]
  ├── 浏览 Banner（首单优惠信息）
  ├── 查看服务项目（建立认知）
  └── 推荐师傅列表 → 点击感兴趣的师傅
  │
  ▼
[师傅主页]
  ├── 查看评价/服务项目/价格
  └── 点击"立即预约"
  │
  ▼
[下单页]
  ├── 选择服务项目（默认选中师傅的第一个项目）
  ├── 选择预约时间（默认"尽快上门"）
  ├── 填写服务地址（搜索医院/家庭/地图选点）
  ├── 填写患者情况（痛点信息收集）
  ├── 确认价格明细
  └── 点击"确认支付"
  │
  ▼
[微信支付调起] → 支付成功
  │
  ▼
[订单详情页]（待接单状态）
  ├── 等待师傅接单（15分钟倒计时）
  └── 推送通知 → 师傅已接单
  │
  ▼
[订单追踪] → 师傅出发 → 查看实时位置 → 师傅到达 → 服务完成 → 评价
```

### 5.2 核心操作反馈清单

| 操作 | 触发反馈 | 成功反馈 | 失败反馈 |
|------|---------|---------|---------|
| 点击"立即预约" | 按钮 Loading + 防重复点击 | 跳转下单页 | Toast "网络异常，请重试" |
| 提交订单 | 按钮 Loading "正在提交..." | 跳转支付 | Toast + 错误原因 |
| 支付 | 调起微信支付 | 跳转订单详情 + 推送 | 支付失败页（含重试按钮） |
| 取消订单 | 二次确认弹窗 | 退款处理中 + 通知 | Toast "取消失败" |

## 7. 设计规范建议

### 7.1 组件建议清单

为保持设计统一性，建议建立以下组件库：

| 组件 | 变体 | 说明 |
|------|------|------|
| **按钮** | 主按钮（深蓝填充）/ 次按钮（白色描边）/ 文字按钮 / 禁用态 / Loading态 | 主按钮用于核心CTA，次按钮用于次要操作 |

| **Loading** | 全屏Loading / 骨架屏 / 行内Loading / 按钮Loading | 骨架屏优先，避免菊花转 |
| **弹窗** | 确认弹窗 / 操作菜单 / 分享弹窗 / 输入弹窗 | 统一遮罩层和圆角规范 |

| **Tab** | 顶部Tab / 底部导航Tab | 统一活跃态指示器样式 |

### 7.2 间距与布局规范

| 层级 | 尺寸 | 用途 |
|------|------|------|
| 页面安全边距 | 16px | 页面左右留白 |
| 卡片内边距 | 14px / 16px | 卡片内容与边框间距 |
| 卡片间距 | 8px / 12px | 卡片之间的垂直间距 |
| 模块间距 | 16px / 20px | 不同模块之间的间距 |

| 列表行高 | 48px / 52px / 56px | 列表项单行高度 |

### 7.3 图标风格建议

- 使用 **线性图标**（线条感），与"洁净""专业"品牌调性一致
- 线条宽度统一：2px
- 圆角端点：round
- 图标尺寸统一：20px / 24px / 28px（三级）
- 推荐使用微信小程序原生 iconfont 或 SVG 图标组件

## 9. 异常与边界场景设计要点

### 9.1 网络异常处理

| 场景 | 设计策略 |
|------|---------|
| **弱网/断网** | 页面不白屏，展示缓存内容（如有）+ 顶部提示条"网络不太稳定" |
| **请求超时** | 3秒内无响应展示 Loading；超时→错误态页面 + "点击重试" |
| **支付中断** | 支付失败页 + 提示"支付未完成，请在30分钟内重新支付" + 倒计时 |

### 9.2 权限异常处理

| 场景 | 设计策略 |
|------|---------|
| **定位被拒** | 首页展示"手动选择城市"引导条，不阻断浏览 |
| **手机号授权被拒** | 下单时提示"需要手机号才能联系您"，引导授权 |
| **相册/相机被拒** | 评价图片上传提示"请在设置中开启相册权限" |

### 9.3 数据边界场景

| 场景 | 设计策略 |
|------|---------|
| **师傅今日约满** | 师傅主页预约按钮置灰 + 标签"今日已约满" + 提示"可预约明日" |
| **师傅当前不可约** | 师傅卡片展示"休息中"状态，卡片半透明或置灰 |
| **城市无师傅** | 首页 Banner 区域展示城市未开通状态 + "预约提醒"订阅 |
| **订单超时未接单** | 推送通知 + 全额退款 + 推荐其他师傅 |
| **评价为空** | 师傅主页评价区展示"暂无评价，先预约体验吧" |
| **收藏列表为空** | 插画 + "还没有收藏的师傅" + 推荐师傅入口 |

### 9.4 微信小程序特殊场景

| 场景 | 设计策略 |
|------|---------|
| **首次启动（未登录）** | 微信静默登录（unionId），不展示登录页 |
| **小程序分享** | 分享卡片：品牌 Logo + "我想洗头 - 专业上门洗头" |
| **从分享链接打开** | 直接进入对应师傅主页/文章详情，无中间页 |
| **小程序被切换到后台** | 回到前台时刷新订单状态 |
| **购买后未支付退出** | 30分钟内可到订单列表找到未支付订单继续支付 |
| **服务超时/未按时开始** | 系统自动处理流程，用户端状态自动更新 |
